# Vigi-Gate Backend

Spring Boot backend for **Vigi-Gate**, a web-based visitor management system for office/residential complexes.

## Core Features
- Visitor check-in (multipart form with portrait upload)
- Active session lookup and checkout
- Risk scoring engine with signal-based rules
- Admin auth (JWT) and role/permission authorization
- Dashboard, visitor detail, and reports APIs
- OpenAI-assisted portrait quality and report narrative generation

## Tech Stack
- Java 21
- Spring Boot 4.0.5
- Spring Security + JWT
- Spring Data JPA (Hibernate)
- PostgreSQL

## Project Structure
- `src/main/java/com/example/berijalanassesment/controller` REST APIs
- `src/main/java/com/example/berijalanassesment/service` business logic
- `src/main/java/com/example/berijalanassesment/models` JPA entities
- `src/main/java/com/example/berijalanassesment/repository` data access
- `src/main/java/com/example/berijalanassesment/security` authn/authz
- `src/main/java/com/example/berijalanassesment/seed` dev seeding
- `docs/` API contracts, screen specs, ERD, risk rules

## Prerequisites
- JDK 21
- PostgreSQL running

## Environment
Copy `.env.example` to `.env` and set values.

Important keys:
- `SPRING_PROFILES_ACTIVE=dev`
- `DB_URL`, `DB_USERNAME`, `DB_PASSWORD`
- `SECURITY_JWT_SECRET`
- `OPENAI_API_KEY` (required for OpenAI-powered summaries/quality scoring)
- `APP_SEED_ENABLED`, `APP_SEED_WIPE`
- `APP_SEED_BULK_VISITORS`, `APP_SEED_BULK_OPEN_ALERTS`, `APP_SEED_BULK_REPORTS`

## Run
```bash
./mvnw spring-boot:run
```

## Dev Seeding
To reseed from scratch:
1. Set `APP_SEED_WIPE=true`
2. Run app once
3. Set `APP_SEED_WIPE=false` for normal subsequent runs

## Auth Flow (Admin)
1. Login via admin auth endpoint to get JWT.
2. Use `Authorization: Bearer <token>` for protected endpoints.

## Key Endpoints
- `POST /api/v1/checkins` (multipart visitor check-in)
- `GET /api/v1/visitors/{visitorId}/active-session` (public)
- `POST /api/v1/checkouts` (public)
- `GET /api/v1/visitors` (admin list with pagination/query/filter)
- `GET /api/v1/sessions/{sessionId}/detail`
- `GET /api/v1/reports/daily`
- `GET /api/v1/reports/daily/intelligence`
- `POST /api/v1/reports/daily/generate`
- `POST /api/v1/reports/daily/export-pdf`
- `GET /api/v1/reports/daily/{reportId}/download?format=pdf`

## Notes
- `timeframe=today` and visitor table time display use `Asia/Jakarta`.
- Report PDF download URL is local (`localhost`) and generated by backend.
